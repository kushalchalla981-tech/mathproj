Screen design must support three main use contexts. GUI preview, parameter adjustment, export. Batch and CLI flows remain terminal driven and do not require GUI surfaces. The GUI should remain minimal and optimized for quick experimentation with compression strength and algorithm modes.

---

SCREEN LIST

1. Launch / Home
2. Image Workspace
3. Compression Controls Panel
4. Export Dialog
5. Batch Report Viewer
6. Settings
7. Error / Unsupported Image Screen

---

HOME SCREEN

Purpose
Entry point. Allows the user to load an image quickly.

Layout hierarchy

Top bar
• App name
• Settings icon

Main body centered vertically

Primary action card
• Large upload area
• Drag and drop region
• “Open Image” button

Secondary area

Recent files list
• Thumbnail
• File name
• Last modified time

Footer

• Supported formats note (JPEG, PNG)

Component breakdown

• Drag and drop upload component
• File picker button
• Recent file list
• Top navigation bar

Interaction logic

• Drag image into drop zone → open workspace
• Click Open Image → file picker → workspace
• Click recent file → workspace loads image

Empty state

Recent files
Text: “No images opened yet”

Error states

Invalid format
Message:
“Unsupported format. Use JPEG or PNG.”

Corrupt file
Message:
“Image cannot be decoded.”

Comparison features

• Side by side layout
• Optional swipe comparison slider

Summary indicators

• File size before
• File size after
• SSIM metric
• Processing time

Interaction logic

• Changing compression slider triggers recomputation
• Preview updates live

Preview recomputation flow

1. User adjusts compression strength
2. Image is mean centered
3. PCA computed
4. Projection reconstructed
5. Preview refreshed

• Option to sync zoom between panels

SCALABILITY CONSIDERATIONS

The layout supports future expansion without redesign.

Possible additions

• Multi component visualization
• Hybrid compression mode
• Tile boundary visualization
• Histogram of eigenvalues
• Quality heatmaps

Controls panel can expand vertically without affecting workspace.

Preview layout supports adding a third comparison mode such as swipe or overlay.
